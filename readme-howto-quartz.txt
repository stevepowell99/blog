Quartz notes (this repo)

What folders mean
- content/: your Markdown pages (Quartz reads these)
- quartz/static/: “raw” static files copied through as-is and served at /static/...
- public/: build OUTPUT (auto-generated by `npx quartz build`). Don’t edit by hand and don’t commit it.

Local dev (Windows / PowerShell)
- Start server (pick any free port):
  npx quartz build --serve --port 3000
  (If you get EADDRINUSE, choose another port e.g. 3001)

Authoring content
- New page: add a .md file under content/ (subfolders become URL paths)
- Page metadata: use YAML frontmatter:
  ---
  title: My Title
  date: 2025-12-27
  tags:
    - tag1
    - tag2
  ---
- Internal links:
  - link to a page: /some-slug (usually derived from filename)
  - link to a tag page: /tags/tagname

Static files / downloads / HTML (IMPORTANT)
- If you want a file reachable at https://pogol.net/static/<path>, put it in:
  quartz/static/<path>
- Examples:
  - quartz/static/tenses/tensesList.pdf  ->  /static/tenses/tensesList.pdf
  - quartz/static/_old_guide/index.html  ->  /static/_old_guide/
- If linking to a “raw HTML page” under /static/ (like slides.html), bypass Quartz SPA router:
  Use HTML in markdown:
    <a href="/static/slides.html" data-router-ignore>slides</a>

Git + deploy (Netlify)
- Default branch is main.
- Normal workflow:
  git status
  git add .
  git commit -m "Describe change"
  git push
- Netlify settings:
  - Base directory: (empty) or .
  - Build command: npx quartz build
  - Publish directory: public
- Netlify auto-deploys on every push to main.

Custom domain (pogol.net)
- Netlify should have:
  - Primary custom domain: www.pogol.net (recommended)
  - Also add apex: pogol.net
- DNS at domaindiscount24:
  - Keep all MX/TXT email records
  - Keep survey.pogol.net A record
  - www.pogol.net CNAME -> pogol-blog.netlify.app (or whatever Netlify shows)
  - pogol.net: ALIAS/ANAME -> apex-loadbalancer.netlify.com (preferred)
    or fallback A -> 75.2.60.5
